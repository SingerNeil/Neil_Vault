# 计算机组成原理考试做题小抄

用法：考试时先判断题型，再套下面步骤。不要从知识点开始想。

## 0. 总做题顺序

1. 先圈关键词：`CPI`、`IEEE754`、`MIPS fields`、`beq/bne`、`datapath`、`pipeline`、`cache address format`。
2. 所有地址题先写单位：byte address 还是 word address。
3. 所有 MIPS 题先判断格式：R / I / J。
4. 所有 cache 题先写三行：address bits、block offset bits、cache blocks / sets。
5. 简答题不要写散文，按固定模板分点答。

## 1. 性能计算题

### 1.1 CPU Time

题目给 `IC`、`CPI`、`clock rate`：

```text
CPU Time = IC * CPI / Clock Rate
```

题目给 `clock cycle time`：

```text
CPU Time = IC * CPI * Clock Cycle Time
Clock Rate = 1 / Clock Cycle Time
```

### 1.2 求 CPI

题目给 execution time、instruction count、cycle time：

```text
CPI = Execution Time / (IC * Cycle Time)
```

题目给 execution time、instruction count、clock rate：

```text
CPI = Execution Time * Clock Rate / IC
```

### 1.3 平均 CPI

```text
Average CPI = total cycles / total instructions
Average CPI = sum(IC_i * CPI_i) / sum(IC_i)
```

### 1.4 比较两个编译器/处理器

执行时间相同，比较 clock rate：

```text
ClockRate_A / ClockRate_B = (IC_A * CPI_A) / (IC_B * CPI_B)
```

如果 `IC_A = IC_B`：

```text
ClockRate_A / ClockRate_B = CPI_A / CPI_B
```

### 1.5 Speedup

```text
Speedup = Old Time / New Time
New Time = Old Time / Speedup
```

判断谁快：CPU time 小的快，不要只看 clock rate。

### 3.2 C 程序到机器执行

按顺序写：

```text
preprocessing -> compiler -> assembler -> linker -> loader -> processor executes machine instructions
```

展开一句：

```text
C source code is compiled into assembly, assembled into object code, linked into an executable, loaded into memory, and finally fetched, decoded and executed by the processor.
```

### 3.3 Memory Hierarchy Management

答题骨架：

```text
Registers -> L1/L2/L3 Cache -> Main Memory -> Disk
```

必写四点：

- 越靠近 CPU：fast、small、expensive。
- 越远离 CPU：slow、large、cheap。
- 靠 temporal locality 和 spatial locality。
- 管理策略包括 placement、identification/tag、replacement、write policy。

## 4. 数制与定点数题

### 4.1 Unsigned hex subtraction

直接按十六进制相减，不看符号位。

例：

```text
0x5ED4 - 0x07A4 = 0x5730
```

### 4.2 Sign-magnitude

步骤：

1. 最高位是 sign。
2. sign = 0 表示正，sign = 1 表示负。
3. 后面位是 magnitude。
4. 先按数学值算，再转回 sign-magnitude。

例：`0x5ED4` 和 `0x07A4` 最高位都是 0，所以：

```text
+0x5ED4 - +0x07A4 = +0x5730
sign-magnitude result = 0x5730
```

### 4.3 Two's complement 转十进制

最高位 0：

```text
直接转十进制
```

最高位 1：

```text
value = unsigned_value - 2^n
```

例：

```text
0x0C000000 最高位是 0
decimal = 0x0C000000 = 201326592
```

## 5. IEEE 754 Single 题

### 5.1 字段

```text
sign: 1 bit
exponent: 8 bits
fraction: 23 bits
bias = 127
```

正规数：

```text
value = (-1)^S * (1.F) * 2^(E - 127)
```

### 5.2 Hex 转 float

步骤：

1. Hex 转 32-bit binary。
2. 切字段：S / E / F。
3. 算真实指数：`E - 127`。
4. 写成：`(-1)^S * 1.F * 2^(E-127)`。

例：

```text
0x0C000000
binary = 00001100 00000000 00000000 00000000
S = 0
E = 00011000b = 24
F = 0
value = 1.0 * 2^(24-127) = 2^-103 ≈ 9.86 * 10^-32
```

### 5.3 十进制转 IEEE754

步骤：

1. 确定符号位。
2. 把绝对值转二进制。
3. 规格化成 `1.xxx * 2^e`。
4. exponent field = `e + 127`。
5. fraction 取小数点后的位，不够补 0。

```text
addr 0: AB

```

Little-endian：最低有效字节放低地址。


## 7. MIPS 编码题

### 7.1 必背寄存器编号

```text
$zero = 0
$v0 = 2
$v1 = 3
$a0-$a3 = 4-7
$t0-$t7 = 8-15
$s0-$s7 = 16-23
$t8-$t9 = 24-25
$sp = 29
$ra = 31
```

常用：

```text
$t0=8, $t1=9, $t2=10
$s0=16, $s1=17, $s2=18, $s3=19, $s4=20
```

### 7.2 R-format

```text
op | rs | rt | rd | shamt | funct
6    5    5    5     5       6
```

常见：

```text
add rd, rs, rt   op=0 funct=32=0x20
sub rd, rs, rt   op=0 funct=34=0x22
and rd, rs, rt   op=0 funct=36=0x24
or  rd, rs, rt   op=0 funct=37=0x25
slt rd, rs, rt   op=0 funct=42=0x2A
sll rd, rt, sh   op=0 rs=0 funct=0
```

### 7.3 I-format

```text
op | rs | rt | immediate
6    5    5      16
```

常见：

```text
addi rt, rs, imm     op=8
lw   rt, offset(rs)  op=35=0x23
sw   rt, offset(rs)  op=43=0x2B
beq  rs, rt, offset  op=4
bne  rs, rt, offset  op=5
lui  rt, imm         op=15=0x0F
ori  rt, rs, imm     op=13=0x0D
```

### 7.4 J-format

```text
op | address
6      26

j   target  op=2
jal target  op=3
```

### 7.5 汇编转机器码

步骤：

1. 判断 R/I/J。
2. 写字段名。
3. 查寄存器编号。
4. 查 op / funct。
5. immediate 转 16 bits，address 转 26 bits。
6. 拼 32 bits。
7. 每 4 bits 转 1 hex。

例：

```text
sw $t1, 32($t2)
op = sw = 43 = 101011
rs = $t2 = 10 = 01010
rt = $t1 = 9  = 01001
imm = 32 = 0000000000100000

101011 01010 01001 0000000000100000
= 0xAD490020
```

### 7.6 机器码转汇编

步骤：

1. 看前 6 bits op。
2. op = 0：R-format，看 funct。
3. op != 0：I-format 或 J-format。
4. 把寄存器编号翻译成寄存器名。

例：

```text
0000 0010 0001 0000 1000 0000 0010 0000
= 000000 10000 10000 10000 00000 100000
op=0, rs=$s0, rt=$s0, rd=$s0, shamt=0, funct=32
= add $s0, $s0, $s0
```

### 7.7 给字段写指令

例：

```text
op=0, rs=3, rt=2, rd=3, shamt=0, funct=34
op=0 -> R-type
funct=34 -> sub
rs=3=$v1, rt=2=$v0, rd=3=$v1
instruction = sub $v1, $v1, $v0
binary = 000000 00011 00010 00011 00000 100010
```

## 8. MIPS 地址计算 / Lab3

### 8.1 `lw/sw`

```text
lw rt, offset(rs)
sw rt, offset(rs)
Effective Address = Reg[rs] + sign_extend(offset)
```

### 8.2 `beq/bne`

```text
Branch Target = PC + 4 + (sign_extend(immediate) << 2)
immediate = (Target - (PC + 4)) / 4
```

注意：branch 里存的是 offset，不是完整地址。

### 8.3 `j`

```text
Jump Target = { (PC+4)[31:28], address field, 00 }
```

如果题目已经给 `address field = 0x100018`：

```text
target = 0x00400060
machine code = 0x08100018
```

### 8.4 `lui/ori`

```text
lui $s1, 0x1234      -> $s1 = 0x12340000
ori $s1, $s1, 0x5678 -> $s1 = 0x12345678
addi $s1, $s1, 1     -> $s1 = 0x12345679
```

### 8.5 Lab3 填空题顺序

看到 Lab3 这种题，按这个顺序填：

1. 先列 data 段变量地址和值。
2. `la` 后确定 `$t0` 指向哪个变量。
3. `lw`：把 memory 中的 word 读到寄存器。
4. `lui/ori/addi`：直接算寄存器值。
5. `bne/beq`：先判断是否跳，再用公式算目标。
6. `j`：用 J-format 公式算目标。
7. `sw`：先算地址，再写入值。
8. `jr $ra`：PC 变成 `$ra`。

常见值：

```text
g = 12 = 0x0000000C
h = 13 = 0x0000000D
i = 24 = 0x00000018
k = 25 = 0x00000019

k=25: i != k -> branch to Else -> f = g - h = -1 = 0xFFFFFFFF
k=24: i == k -> no branch -> f = g + h = 25 = 0x00000019
```

## 9. ALU 实验题

已知：

```text
A = 55H
W = 33H
```

功能表：

| S2 S1 S0 | 功能 | 结果 |
|---|---|---|
| 000 | A + W | 88H |
| 001 | A - W | 22H |
| 010 | A OR W | 77H |
| 011 | A AND W | 11H |
| 100, C=0 | A + W + C | 88H |
| 100, C=1 | A + W + C | 89H |
| 101, C=0 | A - W - C | 22H |
| 101, C=1 | A - W - C | 21H |
| 110 | ~A | AAH |
| 111 | A | 55H |

答解释：

```text
寄存器 A/W 写入需要 CK rising edge；ALU 是组合逻辑，A、W 和控制信号稳定后立即输出结果。
```

## 10. 单周期 Datapath / Homework4

### 10.1 控制信号表

| 指令 | RegDst | ALUSrc | MemtoReg | RegWrite | MemRead | MemWrite | Branch | ALUOp |
|---|---|---|---|---|---|---|---|---|
| R-type | 1 | 0 | 0 | 1 | 0 | 0 | 0 | 10 |
| lw | 0 | 1 | 1 | 1 | 1 | 0 | 0 | 00 |
| sw | X | 1 | X | 0 | 0 | 1 | 0 | 00 |
| beq | X | 0 | X | 0 | 0 | 0 | 1 | 01 |
| addi | 0 | 1 | 0 | 1 | 0 | 0 | 0 | 00 |

### 10.2 各指令数据流

R-type：

```text
Instruction Memory -> Register File(rs,rt) -> ALU -> write rd
```

lw：

```text
Register File reads rs
Sign-extend offset
ALU calculates Reg[rs] + offset
Data Memory read
write rt
```

sw：

```text
Register File reads rs and rt
ALU calculates Reg[rs] + offset
Data Memory write Reg[rt]
```

beq：

```text
Register File reads rs and rt
ALU subtracts / compares
if Zero and Branch: PC = PC + 4 + (offset << 2)
```

### 10.3 LWI 扩展题

题目：

```text
LWI Rt, Rd(Rs)
Reg[Rt] = Mem[Reg[Rd] + Reg[Rs]]
```

已有模块可复用：

```text
Instruction Memory
Register File
ALU
Data Memory
MemtoReg write-back path
PC update logic
main control
```

新需求：

```text
普通 lw: address = Reg[Rs] + sign-extend(immediate)
LWI:     address = Reg[Rs] + Reg[Rd]
```

答题关键：

- ALU 仍然做 add。
- Data Memory 仍然按 ALU result 读。
- 写回仍然写 Rt。
- 需要让 Register File 第二个 read address 可以选择 `rd field`。
- 在第二个 read-register address 入口前加 mux。
- 控制单元增加 LWI 的译码规则。

LWI 控制信号核心：

```text
RegWrite = 1
MemRead = 1
MemWrite = 0
MemtoReg = 1
ALUSrc = 0
ALUOp = add
Write register = Rt
Second source register = Rd
```

## 11. 流水线题 / Lab2

### 11.1 五级流水线

```text
IF -> ID -> EX -> MEM -> WB
```

| 阶段 | 做什么 |
|---|---|
| IF | 取指，PC+4 |
| ID | 译码，读寄存器 |
| EX | ALU 运算 / 地址计算 |
| MEM | 访存 |
| WB | 写回寄存器 |

### 11.2 Pipeline Register

答题：

```text
Pipeline registers are temporary registers between adjacent pipeline stages. They hold instruction, PC, register values, immediate values and control signals so that different instructions can be processed in different stages during the same clock cycle.
```

四个：

```text
IF/ID, ID/EX, EX/MEM, MEM/WB
```

### 11.3 画 cycle diagram

无 hazard：

```text
I1: IF ID EX MEM WB
I2:    IF ID EX MEM WB
I3:       IF ID EX MEM WB
```

做题步骤：

1. 每条指令先斜着排 IF-ID-EX-MEM-WB。
2. 遇到 data hazard，看后指令读寄存器时前指令是否已经写回。
3. 不能 forwarding 就插 stall。
4. branch/jump 看是否需要 flush 或 delay slot。
5. 问第 N cycle 哪条在哪个阶段，就竖着读第 N 列。

### 11.4 三类 hazard

Structural hazard：

```text
same hardware resource needed by two instructions in same cycle
solution: add hardware or stall
```

Data hazard：

```text
later instruction needs result not written back yet
solution: forwarding or stall
```

Control hazard：

```text
branch/jump changes PC
solution: stall, flush, branch prediction, earlier branch decision
```

Load-use hazard：

```text
lw result is available after MEM
next instruction using it usually needs one stall even with forwarding
```

### 11.5 Single-cycle / Multi-cycle / Pipeline 比较

Single-cycle：

```text
simple control; one instruction per long cycle; cycle time determined by slowest instruction
```

Multi-cycle：

```text
instruction split into steps; shorter cycle; functional units can be reused; control more complex
```

Pipeline：

```text
overlap instructions; improves throughput; hazards require stall/forwarding/flush; single instruction latency not necessarily reduced
```

## 12. Cache 地址划分题

### 12.1 通用步骤

1. 主存大小是 `2^m bytes`，则 address bits = `m`。
2. block size 是 `2^b bytes`，则 offset bits = `b`。
3. cache blocks = cache size / block size。
4. direct mapped：index bits = `log2(cache blocks)`。
5. fully associative：没有 index。
6. n-way set associative：sets = cache blocks / n，set index bits = `log2(sets)`。
7. tag bits = address bits - index bits - offset bits。

### 12.2 地址格式

Direct mapped：

```text
Tag | Index | Block Offset
```

Fully associative：

```text
Tag | Block Offset
```

Set associative：

```text
Tag | Set Index | Block Offset
```

### 12.3 Homework5 例题

题目：

```text
Main memory = 64KB
Cache = 256B
Block size = 8B
```

先算：

```text
64KB = 2^16 bytes -> address bits = 16
8B = 2^3 bytes -> offset = 3
cache blocks = 256 / 8 = 32 = 2^5
```

Direct mapped：

```text
index = 5
tag = 16 - 5 - 3 = 8
format = Tag(8) | Index(5) | Offset(3)
```

Fully associative：

```text
index = 0
tag = 16 - 3 = 13
format = Tag(13) | Offset(3)
```

2-way set associative：

```text
sets = 32 / 2 = 16 = 2^4
set index = 4
tag = 16 - 4 - 3 = 9
format = Tag(9) | Set index(4) | Offset(3)
```

## 13. Cache 策略题

### 13.1 Four Questions

```text
Q1 Where can a block be placed?        placement
Q2 How is a block found?               tag / index / valid bit
Q3 Which block should be replaced?     replacement policy
Q4 What happens on a write?            write policy
```

### 13.2 Mapping

Direct mapped：

```text
one memory block -> one cache location
index = block address mod number of cache blocks
fast but conflict miss can be high
```

Fully associative：

```text
one memory block -> any cache line
no index
less conflict miss but expensive search
```

Set associative：

```text
one memory block -> one set, any way inside that set
balance between direct and fully associative
```

### 13.3 Write policy

Write-through：

```text
write cache and memory at the same time
simple, consistent, slower
```

Write-back：

```text
write cache first, write memory when block is replaced
needs dirty bit, faster usually
```

Write-allocate：

```text
on write miss, bring block into cache
```

No-write-allocate：

```text
on write miss, write directly to memory
```

### 13.4 Cache performance

```text
Miss rate = misses / accesses
Hit rate = hits / accesses
```

带 miss penalty 的 CPI：

```text
CPI = ideal CPI + stalls per instruction
stalls per instruction = memory access per instruction * miss rate * miss penalty
```

## 14. Lab4 结论题

Cache capacity 增大：

```text
miss rate usually decreases
reason: more working set fits in cache; capacity/conflict misses decrease
```

Associativity 增大：

```text
miss rate usually decreases
reason: conflict misses decrease
but benefit becomes smaller when associativity is already high
```

Block size 增大：

```text
can reduce compulsory misses by spatial locality
too large block reduces number of cache lines and may increase conflict/capacity misses
so there is a trade-off
```

答题句式：

```text
Cache performance depends on locality, capacity, associativity and block size. Larger capacity and higher associativity usually reduce miss rate, but increase hardware cost and may affect hit latency. Block size should balance spatial locality and cache line count.
```
