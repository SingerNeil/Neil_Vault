现在状态是对的，只差把默认 Ubuntu 源限制为 `amd64/i386`，避免它去 `archive.ubuntu.com` 拉不存在的 `armhf` 索引。你刚才的 `apt-cache policy` 已经说明 `ports.ubuntu.com` 的 `armhf` 包源是通的。

按这个顺序继续。

**1. 修正默认 Ubuntu 源的架构范围**

先备份：

```bash
sudo cp /etc/apt/sources.list.d/ubuntu.sources /etc/apt/sources.list.d/ubuntu.sources.bak
```

然后给这个文件的两个源段都加上 `Architectures: amd64 i386`：

```bash
sudo sed -i '/^Signed-By:/a Architectures: amd64 i386' /etc/apt/sources.list.d/ubuntu.sources
```

检查一下结果：

```bash
cat /etc/apt/sources.list.d/ubuntu.sources
```

你应该看到大致类似这样，两段里都有一行：

```text
Architectures: amd64 i386
```

然后再更新索引：

```bash
sudo apt update
```

这一步不应该再出现 `archive.ubuntu.com ... binary-armhf ... 404` 了。

**2. 安装 ARMv7 交叉编译依赖**

```bash
sudo apt install -y \
  cmake \
  pkg-config \
  gcc-arm-linux-gnueabihf \
  g++-arm-linux-gnueabihf \
  libc6-dev:armhf \
  libssl-dev:armhf \
  zlib1g-dev:armhf
```

装完后检查关键文件：

```bash
command -v arm-linux-gnueabihf-gcc
ls /usr/lib/arm-linux-gnueabihf/libssl.so
ls /usr/lib/arm-linux-gnueabihf/libcrypto.so
```

这三个正常，就说明依赖齐了。

**3. 交叉编译 ARMv7 版工具**

```bash
cd /home/eds/code/Pulsar-Upgrade-Tool
rm -rf build-armv7
cmake -S . -B build-armv7 \
  -DCMAKE_C_COMPILER=arm-linux-gnueabihf-gcc \
  -DOPENSSL_INCLUDE_DIR=/usr/include \
  -DOPENSSL_SSL_LIBRARY=/usr/lib/arm-linux-gnueabihf/libssl.so \
  -DOPENSSL_CRYPTO_LIBRARY=/usr/lib/arm-linux-gnueabihf/libcrypto.so
cmake --build build-armv7 -j
```

验证产物架构：

```bash
file build-armv7/pulsar-flash
```

你要看到的是 ARM 32 位，比如：

```bash
ELF 32-bit LSB pie executable, ARM, EABI5 ...
```

**4. 传到相机并烧录**

```bash
scp build-armv7/pulsar-flash mcu1.bin mcu2.bin root@<camera-ip>:/tmp/
ssh root@<camera-ip>
chmod +x /tmp/pulsar-flash
/tmp/pulsar-flash
```

如果这里打印 usage，就说明程序能启动。然后找串口：

```bash
ls /dev/ttyUSB* /dev/ttyACM*
```

假设是 `/dev/ttyUSB0`，执行：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu1.bin -m 1
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu2.bin -m 2
```

你现在先执行第 1 步和第 2 步。  
如果 `sudo apt update` 还有报错，或者 `apt install` 失败，把那两段输出贴过来，我继续帮你收敛。
