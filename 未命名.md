按这个顺序做就行。你有 `sudo`，那就直接把交叉编译环境装起来，然后编 ARMv7 版本。

**1. 开启 `armhf` 多架构**

```bash
sudo dpkg --add-architecture armhf
sudo apt update
```

执行完后，先确认一下：

```bash
dpkg --print-foreign-architectures
```

你应该能看到：

```bash
armhf
```

**2. 安装交叉编译和 ARMv7 依赖**

直接装这一组：

```bash
sudo apt install -y \
  cmake \
  pkg-config \
  gcc-arm-linux-gnueabihf \
  g++-arm-linux-gnueabihf \
  libc6-dev:armhf \
  libssl-dev:armhf \
  zlib1g-dev:armhf
```

如果这里报某个包不存在，把报错贴出来；大多数 Debian/Ubuntu 系都能装。

安装后检查几项关键内容：

```bash
command -v arm-linux-gnueabihf-gcc
ls /usr/lib/arm-linux-gnueabihf/libssl.so
ls /usr/lib/arm-linux-gnueabihf/libcrypto.so
```

如果这三个都能看到，说明交叉编译环境基本齐了。

**3. 进入项目，重新生成 ARMv7 构建目录**

```bash
cd /home/eds/code/Pulsar-Upgrade-Tool
rm -rf build-armv7
```

**4. 用 CMake 配置 ARMv7 构建**

先用这一条：

```bash
cmake -S . -B build-armv7 \
  -DCMAKE_C_COMPILER=arm-linux-gnueabihf-gcc \
  -DOPENSSL_INCLUDE_DIR=/usr/include \
  -DOPENSSL_SSL_LIBRARY=/usr/lib/arm-linux-gnueabihf/libssl.so \
  -DOPENSSL_CRYPTO_LIBRARY=/usr/lib/arm-linux-gnueabihf/libcrypto.so
```

如果这一步成功，再编译：

```bash
cmake --build build-armv7 -j
```

**5. 验证产物是不是 ARMv7**

```bash
file build-armv7/pulsar-flash
```

你要看到类似这种结果：

```bash
ELF 32-bit LSB pie executable, ARM, EABI5 ...
```

只要不是 `x86-64`，而是 `ARM 32-bit`，就对了。

**6. 如果第 4 步找不到 OpenSSL，用这个备用命令**

有些机器上 `find_package(OpenSSL)` 会优先扫到宿主机的 x86 库，那就改用下面这一条重新配：

```bash
rm -rf build-armv7

PKG_CONFIG_LIBDIR=/usr/lib/arm-linux-gnueabihf/pkgconfig:/usr/share/pkgconfig \
cmake -S . -B build-armv7 \
  -DCMAKE_C_COMPILER=arm-linux-gnueabihf-gcc \
  -DOPENSSL_INCLUDE_DIR=/usr/include \
  -DOPENSSL_SSL_LIBRARY=/usr/lib/arm-linux-gnueabihf/libssl.so \
  -DOPENSSL_CRYPTO_LIBRARY=/usr/lib/arm-linux-gnueabihf/libcrypto.so
```

然后再编：

```bash
cmake --build build-armv7 -j
```

**7. 编好以后传到相机**

假设你的两个固件文件叫 `mcu1.bin` 和 `mcu2.bin`：

```bash
scp build-armv7/pulsar-flash mcu1.bin mcu2.bin root@<camera-ip>:/tmp/
```

**8. 在相机上执行**

SSH 进相机：

```bash
ssh root@<camera-ip>
chmod +x /tmp/pulsar-flash
```

先测试程序能不能启动：

```bash
/tmp/pulsar-flash
```

如果它打印 usage，说明架构没问题，程序能跑。然后找串口：

```bash
ls /dev/ttyUSB* /dev/ttyACM*
```

假设串口是 `/dev/ttyUSB0`，开始烧录：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu1.bin -m 1
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu2.bin -m 2
```
