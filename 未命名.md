现在不是工具问题，是**相机上这个串口设备名不对**。`/dev/ttyUSB1` 在相机里不存在，所以程序打开失败了。

你下一步不要继续重试同一个命令，先把实际串口找出来。

在相机上执行这几条：

```bash
ls -l /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
dmesg | grep -i tty
cat /proc/tty/driver/usbserial 2>/dev/null
```

如果系统比较精简，`dmesg` 可能信息太多，那就用这个更聚焦的：

```bash
dmesg | grep -Ei 'tty|usb'
```

你要找的是类似下面这种设备：

```bash
/dev/ttyUSB0
/dev/ttyUSB1
/dev/ttyACM0
```

大概率会出现两种情况：

1. 板子上实际设备是 `/dev/ttyUSB0`，不是 `/dev/ttyUSB1`
2. USB 串口根本没枚举出来，说明线没接好、驱动没加载，或者设备没被识别

如果你只是想快速验证，先直接试：

```bash
ls /dev/ttyUSB0
```

如果它存在，就改成：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/OnStepX.ino.bin -m 1
```

然后 MCU2 对应地改成：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/SmartWebServer.ino.bin -m 2
```

如果 `/dev/ttyUSB0` 也不存在，那就不是命令写法问题，而是相机当前没识别到 USB-UART 设备。那时重点检查：

```bash
lsusb
dmesg | tail -n 50
```
