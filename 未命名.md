● OnStepX 固件发布 SOP                                                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                                                                                 
  前提条件
  - 本地代码已是最新，所有改动已 commit 并 push 到 main                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                                    
  ---                                                                                                                                                                                                                                                                                                 
  第一步：确认代码是最新的                                                                                                                                                                                                                                                                          
  git checkout main                                                                                                                                                                                                                                                                                   
  git pull                                                                                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                                      
  ---                                                                                                                                                                                                                                                                                               
  第二步：打 tag                         
                                                                                                                                                                                                                                                                                                      
  tag 格式：主版本.次版本.补丁版本，可以加字母后缀（如 a 表示 alpha）
                                                                                                                                                                                                                                                                                                      
  git tag 1.0.0        # 正式版                                                                                                                                                                                                                                                                     
  # 或                                                                                                                                                                                                                                                                                                
  git tag 1.0.0a       # 测试版                                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                                      
  ---                                                                                                                                                                                                                                                                                               
  第三步：推送 tag 到 GitHub             
  git push origin 1.0.0                                                                                                                                                                                                                                                                               
                       
  ▎ 推送 tag 的动作会自动触发 CI，无需手动操作。                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                      
  ---                                    
  第四步：等待编译完成（约 5~10 分钟）                                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                      
  去 GitHub 仓库页面 → 点 Actions → 找到刚触发的 Build Release Firmware → 等绿勾。
                                                                                                                                                                                                                                                                                                      
  ---                                                                                                                                                                                                                                                                                               
  第五步：确认 Release                                                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                      
  去仓库页面 → Releases → 找到对应版本 → Assets 里应有：
                                                                                                                                                                                                                                                                                                      
  OnStepX_1.0.0_esp32.zip   ← 编译好的固件，解压得到 OnStepX.ino.bin                                                                                                                                                                                                                                
  Source code (zip)          ← GitHub 自动生成，忽略即可                                                                                                                                                                                                                                              
  Source code (tar.gz)       ← 同上                                                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                      
  ---                                                                                                                                                                                                                                                                                                 
  如果 CI 失败                                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                      
  去 Actions → 点开失败的 job → 看报错信息，或联系负责 CI 的同事。
                                                                                                                                                                                                                                                                                                      
  ---

  ├──────────────┼────────┤

  ├──────────────┼────────┤
