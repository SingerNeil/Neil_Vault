Optimizing tool selection...我先确认你这台开发机的发行版和 APT 源配置，结果会直接决定为什么 `:armhf` 包找不到，以及该怎么修正。

Ran terminal command:  cat /etc/os-release

Ran terminal command:  grep -R '^deb ' /etc/apt/sources.list /etc/apt/sources.list.d 2>/dev/null | cat

Ran terminal command:  apt-cache policy libc6-dev:armhf libssl-dev:armhf zlib1g-dev:armhf | cat

问题已经缩小到 APT 源配置了；Ubuntu 24.04 正常是能看到这些 `armhf` 包的，我现在检查 deb822 `.sources` 文件里有没有把架构限制死。

Ran terminal command:  ls /etc/apt/sources.list.d/*.sources 2>/dev/null | cat

Ran terminal command:  for f in /etc/apt/sources.list.d/*.sources; do echo '---' "$f"; cat "$f"; done

Ran terminal command:  apt-cache madison libc6-dev | cat

原因已经看清了：你的开发机是 Ubuntu 24.04，主源现在只让 APT 看到了 `amd64` 包，所以 `:armhf` 开发包根本没进索引。要修这个，不是继续重试 `apt install`，而是先把 `armhf` 的 Ubuntu Ports 源加进来。

按下面做。

**1. 新增 ARMHF 软件源**

执行：

```bash
sudo tee /etc/apt/sources.list.d/ubuntu-armhf.sources >/dev/null <<'EOF'
Types: deb
URIs: http://ports.ubuntu.com/ubuntu-ports/
Suites: noble noble-updates noble-backports noble-security
Components: main restricted universe multiverse
Architectures: armhf
Signed-By: /usr/share/keyrings/ubuntu-archive-keyring.gpg
EOF
```

然后更新索引：

```bash
sudo apt update
```

更新完先检查一下包现在能不能看到了：

```bash
apt-cache policy libc6-dev:armhf libssl-dev:armhf zlib1g-dev:armhf
```

如果这一步有版本信息，说明源已经通了。

**2. 安装 ARMv7 交叉编译依赖**

```bash
sudo apt install -y \
  cmake \
  pkg-config \
  gcc-arm-linux-gnueabihf \
  g++-arm-linux-gnueabihf \
  libc6-dev:armhf \
  libssl-dev:armhf \
  zlib1g-dev:armhf
```

装完后检查关键文件：

```bash
command -v arm-linux-gnueabihf-gcc
ls /usr/lib/arm-linux-gnueabihf/libssl.so
ls /usr/lib/arm-linux-gnueabihf/libcrypto.so
```

这三个都正常，就可以编了。

**3. 交叉编译 ARMv7 版 `pulsar-flash`**

```bash
cd /home/eds/code/Pulsar-Upgrade-Tool
rm -rf build-armv7
cmake -S . -B build-armv7 \
  -DCMAKE_C_COMPILER=arm-linux-gnueabihf-gcc \
  -DOPENSSL_INCLUDE_DIR=/usr/include \
  -DOPENSSL_SSL_LIBRARY=/usr/lib/arm-linux-gnueabihf/libssl.so \
  -DOPENSSL_CRYPTO_LIBRARY=/usr/lib/arm-linux-gnueabihf/libcrypto.so
cmake --build build-armv7 -j
```

然后验证架构：

```bash
file build-armv7/pulsar-flash
```

你要看到类似：

```bash
ELF 32-bit LSB pie executable, ARM, EABI5 ...
```

**4. 传到相机**

```bash
scp build-armv7/pulsar-flash mcu1.bin mcu2.bin root@<camera-ip>:/tmp/
```

**5. 相机上执行**

```bash
ssh root@<camera-ip>
chmod +x /tmp/pulsar-flash
/tmp/pulsar-flash
```

如果这里输出 usage，说明程序能启动。然后：

```bash
ls /dev/ttyUSB* /dev/ttyACM*
```

假设串口是 `/dev/ttyUSB0`，执行：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu1.bin -m 1
/tmp/pulsar-flash -p /dev/ttyUSB0 -f /tmp/mcu2.bin -m 2
```

如果第 1 步 `apt update` 或第 2 步 `apt install` 还有报错，直接把报错贴给我。最关键的是先看这一条的输出：

```bash
apt-cache policy libc6-dev:armhf libssl-dev:armhf zlib1g-dev:armhf
```

有了它，我可以马上判断你的源是不是已经修正成功。
