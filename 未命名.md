这是相机端 **不支持 SFTP subsystem**，不是文件路径问题。你现在这台开发机上的 `scp` 默认走的是 **SFTP 模式**，而相机的 SSH 服务端没有开这个功能。

直接改用 **旧版 SCP 协议** 就行，也就是加 `-O`。

你现在重新执行这条：

```bash
scp -O /home/eds/code/Pulsar-Upgrade-Tool/build-armv7/pulsar-flash \
       /home/eds/code/SmartWebServer/build/esp32.esp32.esp32/SmartWebServer.ino.bin \
       /home/eds/code/OnStepX/build/esp32.esp32.esp32/OnStepX.ino.bin \
       root@192.168.124.167:/tmp/
```

如果这个成功，后面继续：

```bash
ssh root@192.168.124.167
chmod +x /tmp/pulsar-flash
ls -l /tmp/pulsar-flash /tmp/SmartWebServer.ino.bin /tmp/OnStepX.ino.bin
/tmp/pulsar-flash
```

如果 `/tmp/pulsar-flash` 能打印 usage，再烧录：

```bash
/tmp/pulsar-flash -p /dev/ttyUSB1 -f /tmp/SmartWebServer.ino.bin -m 2
/tmp/pulsar-flash -p /dev/ttyUSB1 -f /tmp/OnStepX.ino.bin -m 1
```

如果 `scp -O` 还是不行，就直接走 `ssh + cat`，这招最原始但稳定：

```bash
cat /home/eds/code/Pulsar-Upgrade-Tool/build-armv7/pulsar-flash | ssh root@192.168.124.167 'cat > /tmp/pulsar-flash'
cat /home/eds/code/SmartWebServer/build/esp32.esp32.esp32/SmartWebServer.ino.bin | ssh root@192.168.124.167 'cat > /tmp/SmartWebServer.ino.bin'
cat /home/eds/code/OnStepX/build/esp32.esp32.esp32/OnStepX.ino.bin | ssh root@192.168.124.167 'cat > /tmp/OnStepX.ino.bin'
```

然后同样执行：

```bash
ssh root@192.168.124.167
chmod +x /tmp/pulsar-flash
ls -l /tmp/pulsar-flash /tmp/SmartWebServer.ino.bin /tmp/OnStepX.ino.bin
/tmp/pulsar-flash
```

你先优先试 `scp -O`。如果还有报错，把那一段输出贴过来。
